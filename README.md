# IEEE KAU Chatbot 🤖

Arabic RAG chatbot for IEEE KAU Student Branch (Female Section).
Built with Groq (free LLM) + multilingual embeddings + FastAPI + Render.

---

## Stack
| Layer | Tool | Cost |
|-------|------|------|
| LLM | Groq (llama-3.1-8b-instant) | Free |
| Embeddings | multilingual-e5-small | Free (local) |
| Vector Store | Local pickle file | Free |
| API | FastAPI | Free |
| Hosting | Render | Free |

**Total cost: $0/month** ✅

## Setup (Step by Step)

### Step 1 — Get your Groq API key
1. Go to https://console.groq.com
2. Sign up (free)
3. Go to API Keys → Create API Key
4. Copy it

### Step 2 — Clone & install
```bash
git clone <your-repo>
cd ieee-chatbot
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### Step 3 — Add your PDF
```
Copy your PDF to:  data/ieee_kau.pdf
```

### Step 4 — (Optional) Add extra knowledge
Create `.txt` files in `data/extra_docs/` for anything not in the PDF:

```
data/extra_docs/events.txt          ← upcoming events
data/extra_docs/faq.txt             ← common questions
data/extra_docs/announcements.txt   ← latest news
```

Example `faq.txt`:
```
س: كيف أنضم للنادي؟
ج: يمكنك الانضمام عبر الرابط: [رابط النموذج]

س: متى تكون اجتماعات النادي؟
ج: تُعقد الاجتماعات كل أسبوعين، يوم الأحد الساعة 5 مساءً.
```

### Step 5 — Build the vector store
```bash
cp .env.example .env
# Edit .env and paste your GROQ_API_KEY

python ingest.py
```
You should see: `🎉 Vector store saved to data/vector_store.pkl`

### Step 6 — Run locally
```bash
uvicorn main:app --reload
```
API is now running at: http://localhost:8000

### Step 7 — Test it
Open `test_chat.html` in your browser — it connects to localhost:8000 by default.
Try asking: "كيف أنضم للنادي؟" or "من هي رئيسة النادي؟"

## Deploy to Render (Free)

1. Push your code to a **GitHub repo**
   - ⚠️ Make sure `.gitignore` is there — don't commit `.env` or `vector_store.pkl`
   - ✅ DO commit `data/ieee_kau.pdf`

2. Go to https://render.com → New → Web Service

3. Connect your GitHub repo

4. Settings:
   - **Build Command:** `pip install -r requirements.txt && python ingest.py`
   - **Start Command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`
   - **Add Disk:** Mount path `/opt/render/project/src/data`, size 1GB

5. Environment Variables → Add:
   - Key: `GROQ_API_KEY`
   - Value: your Groq key

6. Click Deploy 🚀

Your chatbot will be live at: `https://ieee-kau-chatbot.onrender.com`

## Connect to Your Website

Once deployed, your frontend calls ONE endpoint:

```javascript
// Replace with your actual Render URL
const API_URL = "https://ieee-kau-chatbot.onrender.com";

const response = await fetch(`${API_URL}/chat`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    message: "كيف أنضم للنادي؟",
    conversation_history: []   // pass previous messages for context
  })
});

const data = await response.json();
console.log(data.response); // Arabic answer
```

## Updating the Knowledge Base

To add new info (events, announcements, etc.):
1. Edit or add `.txt` files in `data/extra_docs/`
2. Run `python ingest.py` again locally
3. Commit and push → Render auto-redeploys

Google Sheets Events Integration 📊

Instead of manually editing events.txt, the chatbot can read events directly from Google Sheets in real-time.
This allows the club team to add or update events without touching the code.

 1 — Create Google Sheet

Create a Google Sheet with these columns:

event_id	title	description	category	date	time	location	registration_link	announcement_link	is_upcoming	created_at

Example row:
| 1 | AI Workshop | Intro to AI | Workshop | 2026-05-10 | 18:00 | Online | https://form
 | https://x.com
 | TRUE | 2026-04-05 |

 
 2- Backend Integration

The backend reads the sheet as CSV:

import pandas as pd

SHEET_ID = "YOUR_SHEET_ID"

def load_events_from_sheet():
    url = f"https://docs.google.com/spreadsheets/d/{SHEET_ID}/export?format=csv"
    df = pd.read_csv(url)
    return df.to_dict(orient="records")

Now whenever someone asks about events, the chatbot loads events directly from Google Sheets.
